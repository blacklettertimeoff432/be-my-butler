# BMB Demo — Intentional Bugs Reference

Use this file to verify that BMB found and fixed all 6 planted issues.

---

## Bug 1: SQL Injection (CRITICAL)

**Location**: `src/routes/todos.js` — all route handlers
**Problem**: Every SQL query uses string interpolation instead of parameterized queries.
```js
// Vulnerable (current)
db.prepare(`INSERT INTO todos (title) VALUES ('${title}')`).run();

// Fixed
db.prepare('INSERT INTO todos (title) VALUES (?)').run(title);
```
**Impact**: An attacker can inject arbitrary SQL via the title field or URL params.
**Expected Fix**: Replace all template literal queries with `?` placeholders and bound parameters.

---

## Bug 2: No Input Validation

**Location**: `src/routes/todos.js` — POST and PUT handlers
**Problem**: The `title` field is passed directly to the database with no validation.
Accepts empty strings, `null`, `undefined`, objects, arrays — anything.
**Expected Fix**: Validate that `title` is a non-empty string, return 400 on invalid input.

---

## Bug 3: Missing Error Handling

**Location**: `src/index.js` and `src/routes/todos.js`
**Problem**: No try/catch around database operations, no Express error middleware.
A malformed query or DB error crashes the entire server with an unhandled exception.
**Expected Fix**: Add try/catch in route handlers and an Express error middleware in index.js.

---

## Bug 4: Race Condition on PUT

**Location**: `src/routes/todos.js` — PUT handler
**Problem**: `PUT /todos/:id` runs an UPDATE without first checking if the todo exists.
`db.prepare(...).run()` returns `{ changes: 0 }` for non-existent IDs but the handler
ignores this and returns `{ message: 'Updated' }` — misleading the client.
**Expected Fix**: Check `result.changes === 0` and return 404 if the todo doesn't exist.

---

## Bug 5: Zero Tests

**Location**: Project root — no test files, no test script
**Problem**: `package.json` has no `test` script. No test directory exists.
**Expected Fix**: Add a test framework (e.g., vitest, jest, or node:test), write tests
covering all CRUD operations and edge cases (invalid input, 404, SQL injection attempts).

---

## Bug 6: Vulnerable Dependencies

**Location**: `package.json`
**Problem**: express `4.17.1` (released 2019) has known vulnerabilities.
**Expected Fix**: Update to latest express 4.x (4.21+) or express 5.x.
Run `npm audit` to verify resolution.
